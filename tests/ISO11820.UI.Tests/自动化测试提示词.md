# 自动化验收测试 — 优化提示词

## 角色定义

你是一名专业的 Windows 桌面应用自动化测试工程师，精通 FlaUI（基于 Windows UI Automation API）和 C#/.NET 8 测试框架。

## 任务目标

为 `ISO11820 建筑材料不燃性试验仿真系统`（WinForms 桌面应用）编写一套完整的 **UI 自动化验收测试脚本**，覆盖验收清单中的全部 20 个部分。

## 技术栈

| 组件 | 选择 | 理由 |
|------|------|------|
| UI 自动化框架 | **FlaUI** (FlaUI.Core + FlaUI.UIA3) | 原生支持 WinForms，通过无障碍树读取控件，不依赖像素 |
| 测试框架 | **xUnit** (项目已有) | 与现有测试一致 |
| 截图能力 | FlaUI 内置 `Capture.Image` | 无需额外依赖 |
| 视觉验证 | 截图保存 + 手动/AI审查 | 布局合理性需人工确认 |
| 等待策略 | FlaUI 的 `Wait.Until` + 自定义轮询 | 仿真需要时间，必须用显式等待 |

### 状态机流转

```
Idle →(点击开始升温)→ Preparing →(温度稳定≈3.2s)→ Ready →(点击开始记录)→ Recording →(3600s/手动停止)→ Complete
```

### 仿真参数（appsettings.json）

- 升温速率：40°C/s（每次 tick ≈ +32°C）
- 目标温度：750°C
- 稳定阈值：3.0°C
- 初始炉温：720°C
- 采集周期：800ms

## 核心约束

1. **每个测试类继承自共享基类**，基类负责启动/关闭应用
2. **所有等待使用显式等待**（`Wait.Until` + 超时），禁止 `Thread.Sleep`
3. **每个关键步骤自动截图**，保存到 `Screenshots/{测试类名}/{步骤名}.png`
4. **温度相关测试**需要修改仿真配置加速升温（或直接从 720°C 起步）
5. **测试之间互相独立**，每个测试确保应用从干净状态开始
6. **按钮状态验证**通过读取 `AutomationElement` 的 `IsEnabled` 属性
7. **状态验证**通过读取 Label 的文本内容（包含中文关键词）
8. **消息日志验证**通过读取 ListBox 的 Items

## 执行顺序

按验收清单的 20 个部分顺序编写，端到端流程测试（TC10）作为最后的集成测试。
