解决方案由两个 .NET 8 项目组成：`ISO11820.Core`（无 UI 依赖，仅含 `Enums/TestState.cs`、`Models/SystemMessage.cs`、`Models/TemperatureSnapshot.cs`、`Contracts/IRuntimeClock.cs`）被 `ISO11820.App` 引用，形成 Core → App 的单向依赖。

App 层按职责分目录组织：
- `Program.cs` 为 WinExe 入口，调用 `Bootstrapper.Create()` 组装上下文后启动 `MainForm`；
- `App/Iso11820AppContext.cs` 作为 DI 容器替代品，集中持有所有协调器与服务实例；
- `Config/AppSettings.cs` 通过 `appsettings.json` + `System.Text.Json` 加载配置，并提供 `ResolvePaths` 链式解析相对路径；
- `Features/*` 下每个业务域一个 Coordinator（Auth、Calibration、Export、History、TestExecution、TestRecord），对外暴露用例级 API；
- `Infrastructure/Persistence` 使用 `Microsoft.Data.Sqlite` 直接建表（`DatabaseInitializer` + `DbHelper` + `Models/*.cs` POCO）；`FileStorage/CsvSampleWriter.cs` 负责采样数据落盘；
- `Runtime/Controller` 与 `Runtime/Services` 封装数据采集线程（`DaqWorker`）与硬件交互（`SensorSimulator` 模拟模式）；
- `UI/Forms`、`UI/Panels`、`UI/Dialogs`、`UI/Common`、`UI/Chart` 分层组织 WinForms 界面，图表使用 OxyPlot.WindowsForms。

测试分为两层：`tests/ISO11820.Tests` 用 xUnit 对 Coordinator/基础设施做单元测试；`tests/ISO11820.UI.Tests` 基于 FlaUI 驱动真实 WinForms 进程进行端到端验收测试，通过 `RunTests.ps1` 脚本统一编译、运行并收集截图。
