- 业务编排采用 Coordinator 模式：每个 Features 子目录包含一个 XxxCoordinator 类，聚合底层服务并暴露用例级方法，UI 层只依赖 Coordinator。
- 配置对象使用 sealed class + init-only 属性 + ResolvePaths(baseDirectory) 返回新实例的模式，避免可变状态，路径解析统一走 AppSettingsPathResolver。
- 运行时状态以枚举表达（如 TestState: Idle→Preparing→Ready→Recording→Complete），配合 ButtonStateMatrix/ButtonStateManager 将状态映射到按钮可见/可用矩阵。
- 数据库访问不使用 ORM，而是 DbHelper 封装 SQLiteConnection + Models POCO 手动映射，迁移由 DatabaseInitializer.EnsureCreated() 在启动时执行。
- UI 自动化测试用例文件命名遵循 TCNN_描述.cs 约定（TC01_Login 至 TC10_FullFlow），并通过 RunTests.ps1 的 -Filter 参数按前缀筛选执行。
